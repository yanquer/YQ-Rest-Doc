介绍
===================================

使用 Pycharm 时, 当重写父类时, 若定义了不同的参数列表, 会出现警告.

这是因为: 由于增加了参数，从而改变了同父类方法的一致性违反了LSP原则（在使用父类的场景，替换为子类也一样可行）。

解决方式为，在子类重写的方法中为参数里添加默认值赋值，这样就确保了父类方法中定义的参数在子类中一定不会失效从而确保了自上而下的一致性。
